Testability linter

Every requirement is graded testable, vague, or untestable — with a one-click measurable rewrite for the ones that would never survive contact with reality. A spell-checker for product thinking.

⚙

Eval compiler

Testable clauses become runnable checks: deterministic assertions where possible, transparent LLM-judged rubrics where not. Point the suite at any HTTP endpoint and get pass/fail per promise.

◎

Living dashboard

The PRD stops being a dead document. Pass rates per clause, trends across runs, and a top-level promises-kept score — a coverage report for product intent, not code.

Spec → suite → verdict

Eval tools start from datasets. Doc tools stop at prose. Clause connects the two: the PRD is the source of truth, and it compiles.

01 WRITE

Paste your PRD

Plain markdown. Clause segments it into atomic, addressable requirement clauses.

02 LINT

Fix the untestable

Vague promises get flagged with suggested rewrites you can accept in one click.

03 COMPILE

Generate the suite

Each clause becomes eval cases — happy path, edge, and adversarial.

04 RUN

Watch the promises

Run against your AI feature. Failures map back to the exact clause they broke.

"Evals are the new PRD" is 2026's loudest product-management advice — but no tool actually turns a PRD into evals. That's the gap Clause closes.
